Banco de dados relacionais (SQL)

Os fundamentos de banco de dados relacionais (SQL), surgiram na empresa IBM entre as décadas de 1960 a 1970, utiliza como base tabelas compostas por linhas e colunas que podem se relacionar de forma muito parecida com uma tabela do Excel, onde cada banco pode ser considerado uma planilha, cada aba é como uma tabela e cada tabela possuindo linhas e colunas.

Dessa forma, os componentes do banco se tornam um arquivo exclusivo, se relacionando por meio das colunas da tabela, consequentemente esse tipo de ligação gera integridade ao SGBD proporcionando mais consistência ao manusear os dados, que inclusive é uma das suas grandes vantagens, além de possuir bastante confiabilidade, esse tipo de modelo também é usado para rastrear inventários, processar transações de comércio eletrônico, gerenciar grandes quantidades de informações essenciais sobre o cliente, entre outros.

O banco de dados relacionais costuma ser utilizado para reduzir as anomalias e para proteger o seu banco de perda de integridade.
Alguns dos principais bancos de dados relacionais utilizados atualmente são:

  • Microsoft Acces
  • MySQL
  • PostgreSQL
  • SQLServer

Por outro lado, esse modelo de banco de dados possui algumas desvantagens quando se trata de escalabilidade e flexibilidade.

Banco de dados não relacional (NoSQL)

Já o banco de dados não relacional (NoSQL), surgiu como uma alternativa a ocasiões onde seja necessário o armazenamento de uma grande quantidade de dados não estruturado.

O termo NoSQL foi utilizado em 1998 como o nome de um banco de dados não relacional de código aberto, criado por Carlo Strozzi.
Alguns de seus benefícios são:

  • Lidar com grandes volumes de dados em alta velocidade.
  • Armazenar dados não estruturados, semiestruturados ou estruturados.
  • Arquitetura de escalabilidade horizontal.
  • O máximo aproveitamento da nuvem para oferecer tempo de inatividade zero.

As quatro categorias de bancos de dados principais do NoSQL são:

Baseada em documentos, conhecido como banco de dados orientado a documentos, possibilita o armazenamento de milhões de documentos, cada documento possui um ou mais campos, que podem armazenar um conteúdo tipado com uma string, data, inteiro, array, entre outros.

Uma de suas vantagens é a flexibilidade da modelagem, em que cada documento pode possuir campos diferente, como no caso de um super mercado por exemplo, em que podemos ter vários produtos com características diferentes.

Os principais bancos de dados dessa categoria são:

  • MongoDB
  • CouchDB
  • RavenDB
  • Terrastore

Baseada em grafos, é comumente utilizado em cenários de redes sociais online, em que os nós representam as entidades e os laços as interconexões entre eles, possibilitando atravessar o grafo seguindo as relações.

Podemos utilizar esta categoria para lidar com problemas relacionados a lista de controle de acesso e sistema de recomendações.

Os principais bancos de dados dessa categoria são:

  • Neo4j
  • FlokDB
  • GraphDB
  • ArangoDB

Comum Family Store,também conhecido como banco de dados orientado a coluna, seus dados ficam organizados em grupos de colunas.

Esse tipo de banco de dados foi desenvolvido para aumentar a escalabilidade horizontal com a utilização de clusters distribuído de hardware de baixo custo

Os principais bancos de dados dessa categoria são:

  • Hbase
  • Cassandra
  • Hypertable
  • Accumulo

Por fim, algumas características que podem nos ajudar a considerar o uso do banco de dados NoSQL, é Quando:

  • Houver alto volume de dados em grande escala,
  • Os dados forem dinâmicos e frequentemente alterados,
  • For necessário gravação rápida, e a segurança da gravação não for critica.
  • As cargas de trabalho não exigirem garantias de ACID.
  • Os dados exigirem uma ampla distribuição geográfica.

E podemos considerar o uso do banco de dados relacional quando:

  • Garantia de ACID for necessária.
  • Os dados forem previsíveis e altamente estruturados
  • Os usuários forem mais centralizados.
  • A segurança de gravação for um requisito.
  • O volume de dados for consistente.

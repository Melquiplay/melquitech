Requisitos funcionais e não funcionais

Imagem gerada por IA

Na fase de desenvolvimento de um sistema, existem várias etapas muito importante que precisam ser cumpridas, uma delas é o levantamento de requisitos.

Durante o levantamento de requisitos a equipe de desenvolvimento busca entender melhor quais são as necessidades do cliente, e quais objetivos o software deve atender, evitando ao máximo erros de interpretação, pois muitas vezes o cliente descreve o software em seu ponto de vista, mas na cabeça de cada pessoa da equipe pode estar passando uma visão completamente diferente do software.

Então para evitar esse tipo de problema é de extrema importante que o Analista de Requisitos:

  • Entenda as reais necessidades do cliente
  • Crie o modelo de sistema a seguido nas próximas etapas de desenvolvimento
  • Verifique a viabilidade de implantação dessas necessidades
  • Defina funções aos participantes do desenvolvimento
  • Estabeleça cronogramas

Para ajudar a identificar os requisitos, os desenvolvedores podem trabalhar com os stakeholders que são (todos os interessados no sistema)

Domínio de Negócio: Também é muito importante ter o domínio de negócio, que é a área especifica para qual o software será desenvolvido, que pode ser por exemplo pra área médica, área de finanças... então vamos imaginar que eu precise desenvolver um software para um empresa de engenharia civil, mas eu não entendo da área de engenharia civil, então provavelmente eu irei precisar conversar com as pessoas dessa área para conseguir levantar os requisitos e ter o Domínio de Negócio em que aquele software será aplicado.

TIPOS DE REQUISITOS

Os requisitos são classificados em dois grupos, são eles:

  • Requisito Funcionais (RF)
  • Requisitos Não-Funcionais (RNF)

TIPOS DE REQUISITOS

Os Requisitos Funcionais Refere-se basicamente ao que o sistema deve fazer, e sem esses requisitos o sistema não funciona de forma correta, ou até mesmo nem funciona.
Exemplos de Requisitos Funcionais em um sistema acadêmico:

  • Lançamento de notas do aluno
  • Emissão de boletos para pagamento das mensalidades do curso

Exemplos em um sistema de Super Mercado:

  • Adicionar um novo produto ao estoque
  • Calcular o total de vendas
  • Fazer o cancelamento de uma venda
  • Realizar reajustes de preço
  • Consultar o estoque

Requisitos Não-Funcionais (RNF)

Os Requisitos Não-Funcionais indicam características de qualidade que o sistema deve possuir e que estão relacionadas às funcionalidades previstas.
Como por exemplo:

  • Desempenho
  • Escalabilidade
  • Usabilidade
  • Confiabilidade
  • portabilidade

Ou seja, são características que estão mais voltadas ao desempenho do sistema e não as funcionalidades em si.
Os Requisitos Não-Funcionais podem ser classificados em várias categorias, como:

Requisitos de Produto

  • Requisito de usabilidade
  • Requisito de eficiência
  • Requisito de confiabilidade
  • Requisitos de portabilidade

Requisitos Organizacionais

  • Requisitos de entrega
  • Requisitos de implementação
  • Requisitos de padrões

Requisitos Organizacionais

  • Requisitos de interoperabilidade
  • Requisito estéticos
  • Requisitos legais.
